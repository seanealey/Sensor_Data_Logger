The Sensor Data project is a modular and easily modified system for collecting data from an ESP32 and writing it to a CSV file. The system is made up of two main parts: the ESP32 firmware and a Python GUI that runs on a host computer.

The GUI acts as the controller for the ESP32. It detects connected devices, sends commands over serial, and logs incoming sensor readings to CSV files. Once recording starts, timestamps are generated by the ESP32 immediately before the sensor data is transmitted. This avoids timing errors caused by serial latency on the host computer.

The GUI supports multiple ESP32 devices at the same time over separate COM ports. Each device is managed through its own session, allowing simultaneous recordings while still maintaining individual control of each data stream.

Consult the project WIKI for a more detailed description of the project and how to use it.
